microChess — Design System

Tema default “Editorial Claro”. Tokens, tipografia, espaçamento e componentes base reutilizados em todas as telas. O Claude Code deve copiar o bloco :root e os componentes que forem usados — nunca importar este arquivo em runtime.

v1 · default light + dark mobile-first · max 440px
01

Princípios

Guias para decisões futuras. Quando em dúvida, volte aqui.

VelocidadePartida dura poucos minutos. A UI nunca pode ser o gargalo. Alvos de toque grandes (≥44px), transições curtas (≤420ms), confirmação em 1 tap.
ClarezaUma intenção por tela. Hierarquia óbvia: eyebrow · título · ação. Zero ornamento no default (o charme vem dos skins).
Motion como feedbackAnimação serve ao entendimento (onde bater, quem ganhou, o que está pronto). Juicy nos duelos e no game over — sutil no resto.
Densidade equilibradaInformação essencial em primeira vista (fase, timer, placar). Resto acessível via drill-down.
AcessibilidadeContraste AA no mínimo. Foco visível em tudo interativo. Peças em Unicode com contorno, nunca só cor.

05

Motion

Durations + easings nomeados. Juicy onde importa (duelo, game over), sutil no resto.

ease-outEntrada padrão (cards, sheets). cubic-bezier(0.22, 1, 0.36, 1)
ease-springBotões, micro-interações, pop. cubic-bezier(0.34, 1.56, 0.64, 1)
ease-smoothTroca de cor/tema. cubic-bezier(0.4, 0, 0.2, 1)
durationsxs 120 · sm 180 · md 260 · lg 420 · xl 640. Regra: se é feedback imediato de toque, ≤180ms.
timing dos duelosRoll do dado 500ms, reveal do vencedor 260ms, captura 420ms. Total do momento duelo ≤ 1.8s.
